## Phase Goal

**As a** visitor who tapped the sticky cart button, **I want to** see a right-side drawer with every item I've added — brand, name, variant, price, quantity, subtotal — plus a running total and a way to remove items, **so that** I can review my order before copying it to Messenger.

<objective>
Build the cart drawer surface per UI-SPEC §8: a shadcn `<Sheet side="right">` containing header, scrollable item list (or empty state), and footer with totals. Mounted globally in `app/layout.tsx`. Wired to `useCartUiStore` (open state, Plan 04) and `useCartStore` (items, Plan 01).

Purpose: Delivers CART-05, CART-06, CART-08 — the visible drawer surface and per-item removal. The copy-to-Messenger button in the footer is a placeholder slot in this plan; Plan 06 owns the wiring.

Output: A working drawer that opens from the sticky FAB, displays every cart item with full detail, and removes items on demand. Item list is empty (drawer shows empty state) by default since the cart starts empty.

Wave coordination note: This plan runs in Wave 4. It consumes the persisted cart store from Plan 01, the format helpers + `<VariantBadge>` from Plan 02, and the `useCartUiStore` + drawer placeholder slot (the FAB's `openDrawer` wiring) from Plan 04. It does NOT depend on Plan 03 — the product detail surface and the cart drawer surface are independent UI areas. Plan 03 can run in parallel with this plan's prerequisites and merge cleanly.
</objective>

    **1. `components/cart-drawer.tsx`** — The drawer shell. Client Component because it consumes both Zustand stores. Composition per UI-SPEC §8 (header + item list + footer + empty state):
    ```tsx
    'use client';

    import { Sheet, SheetContent, SheetHeader, SheetTitle } from '@/components/ui/sheet';
    import { Button } from '@/components/ui/button';
    import { useCartStore } from '@/lib/cart-store';
    import { useCartUiStore } from '@/lib/cart-ui-store';
    import { getProductById, getVariant } from '@/lib/inventory';
    import { formatPrice, formatItemCount } from '@/lib/format';
    import { CartDrawerEmpty } from '@/components/cart-drawer-empty';
    import { CartDrawerItem } from '@/components/cart-drawer-item';

    export function CartDrawer() {
      const isDrawerOpen = useCartUiStore((s) => s.isDrawerOpen);
      const setDrawerOpen = useCartUiStore((s) => s.setDrawerOpen);
      const items = useCartStore((s) => s.items);
      const isHydrated = useCartStore((s) => s.isHydrated);

      // Resolve display data fresh from inventory on every render (per CONTEXT D-06).
      const resolved = items
        .map((item) => {
          const product = getProductById(item.product_id);
          const variant = product ? getVariant(product, item.variant_id) : undefined;
          if (!product || !variant) return null;
          return { item, product, variant };
        })
        .filter((x): x is { item: typeof items[number]; product: NonNullable<ReturnType<typeof getProductById>>; variant: NonNullable<ReturnType<typeof getVariant>> } => x !== null);

      const totalItems = resolved.reduce((sum, r) => sum + r.item.quantity, 0);
      const totalPrice = resolved.reduce((sum, r) => sum + r.variant.price * r.item.quantity, 0);
      const isEmpty = !isHydrated || resolved.length === 0;

      return (
        <Sheet open={isDrawerOpen} onOpenChange={setDrawerOpen}>
          <SheetContent
            side="right"
            className="w-[85vw] sm:w-[420px] flex flex-col p-0 gap-0"
          >
            <SheetHeader className="px-5 py-4 border-b border-neutral-200">
              <SheetTitle id="cart-title" className="text-xl font-semibold text-neutral-950 text-left">
                Καλάθι
              </SheetTitle>
            </SheetHeader>

            <div className="flex-1 overflow-y-auto px-5 py-4">
              {isEmpty ? (
                <CartDrawerEmpty />
              ) : (
                <ul className="space-y-4">
                  {resolved.map((r) => (
                    <CartDrawerItem
                      key={`${r.item.product_id}::${r.item.variant_id}`}
                      item={r.item}
                      product={r.product}
                      variant={r.variant}
                    />
                  ))}
                </ul>
              )}
            </div>

            <div className="border-t border-neutral-200 px-5 py-4 space-y-3 bg-white">
              <div className="flex items-baseline justify-between">
                <div>
                  <p className="text-sm text-neutral-600">Σύνολο</p>
                  <p className="text-xs text-neutral-600">{formatItemCount(totalItems)}</p>
                </div>
                <p className="text-xl font-semibold text-neutral-950">{formatPrice(totalPrice)}</p>
              </div>
              <Button
                className="w-full"
                disabled
                aria-disabled="true"
                data-slot="copy-cta-placeholder"
              >
                📋 Αντιγραφή για Messenger
              </Button>
            </div>
          </SheetContent>
        </Sheet>
      );
    }
    ```
    Decisions baked in:
    - Width: `w-[85vw] sm:w-[420px]` per UI-SPEC §Responsive Behavior + D-09.
    - Layout: `flex flex-col p-0 gap-0` so the three sections (header / scrollable list / footer) stack to fill height. Override shadcn defaults that add padding.
    - Header: `<SheetHeader>` + `<SheetTitle>` with id `cart-title` — Radix Sheet auto-wires `aria-labelledby` if `SheetTitle` exists; explicit `id="cart-title"` is belt-and-suspenders per UI-SPEC §Drawer interaction states. `text-left` because shadcn's default `<SheetTitle>` style may center on some configurations. Close button (X) is auto-rendered by shadcn `<SheetContent>` — no need to add a manual `<SheetClose>` element unless the X is missing in the shadcn build (in which case add it explicitly with `aria-label="Κλείσιμο"`).
    - Item list: `flex-1 overflow-y-auto px-5 py-4`. When empty → `<CartDrawerEmpty />`; otherwise → `<ul className="space-y-4">` of `<CartDrawerItem>`.
    - Per CONTEXT D-06: resolved display data is computed on every render. If a persisted item points to a missing product/variant (shouldn't happen post-hydrate-cleanup from Plan 01, but defensive), it's filtered out — never crashes the drawer.
