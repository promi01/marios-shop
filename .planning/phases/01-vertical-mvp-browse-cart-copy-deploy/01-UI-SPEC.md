# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1 (Vertical MVP — Browse, Cart, Copy, Deploy).
> Generated by `gsd-ui-researcher`, verified by `gsd-ui-checker`.
> All locked decisions (D-13..D-26) from `01-CONTEXT.md` are honored as-is and elaborated into concrete tokens, anatomy, states, and copy.

## Design System

## Spacing Scale

8-point scale (Tailwind defaults), multiples of 4. Phase 1 uses these tokens exclusively:

| Token | Tailwind | Value | Usage in Phase 1 |
|-------|----------|-------|------------------|
| xs | `1` | 4px | Badge inner gap (icon + text), tight inline padding |
| sm | `2` | 8px | Compact element spacing (badge px in pure form), card image-to-text gap on mobile |
| md | `4` | 16px | Default element spacing — card inner padding, product page section gap, drawer item padding |
| lg | `6` | 24px | Section padding, hero vertical padding, grid gap on tablet+, drawer side padding |
| xl | `8` | 32px | Major layout gaps, product page top padding desktop |
| 2xl | `12` | 48px | Hero vertical padding desktop, page bottom padding |
| 3xl | `16` | 64px | Page-level safe-area bottom (above sticky cart button) |

### Touch Target Exceptions

- **Sticky cart button:** circular, 56×56px (Tailwind `h-14 w-14`) — exceeds the 44×44px iOS minimum. NOT on the 8-point scale because circular FABs follow Material/iOS guidance for thumb reach.
- **Variant "Προσθήκη" button:** min-height 44px (Tailwind `h-11` = 44px) on product page — meets mobile touch target without ceremony.
- **Drawer "Remove" icon button:** 44×44px tap area (visual icon may be smaller but hit-area is full `h-11 w-11`).

### Sticky Cart Button Position

- Bottom-right of viewport, **16px from right edge, 16px from bottom edge** (D-12 says "~16px from edges"), on top of all content. Uses `position: fixed; bottom: 16px; right: 16px; z-index: 40`.
- Drawer (`Sheet`) lives on `z-index: 50` so it covers the sticky button when open.

## Typography

Geist Sans only. Exactly **4 sizes** + **2 weights** for Phase 1.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Caption / Badge | 12px | 400 | 1.4 (≈17px) | `text-xs leading-snug` | Variant badges ("Σφραγισμένο", "Decant"), stock indicators, drawer item count line |
| Body | 14px | 400 | 1.5 (21px) | `text-sm leading-normal` | Tagline, secondary text, drawer item meta (variant + size), product page body copy, description_gr |
| Body-emphasis | 16px | 600 | 1.5 (24px) | `text-base font-semibold` | Brand name on cards, "Σύνολο" label, prices in drawer, primary button label |
| Heading | 20px | 600 | 1.3 (26px) | `text-xl font-semibold` | Product name on product page, drawer header "Καλάθι" |
| Display | 32px | 600 | 1.2 (38px) | `text-3xl font-semibold` | Hero wordmark "Marios Shop" (mobile + tablet); 36–40px on `md:` and above via `md:text-4xl` |

Weights: **regular (400)** and **semibold (600)** only. No `font-medium` (500), no `font-bold` (700) — keeps the minimal "fragrance-brand" aesthetic from CONTEXT.md D-13.

### Line-Height Rules

- Body and meta text: `leading-normal` (1.5) — readability on small screens.
- Headings (≥20px): `leading-tight` to `leading-snug` (1.2–1.3) — tighter, more refined.
- Single-line labels in buttons and badges: line-height does not matter (single line) — use defaults.

### Font Family Declaration

```ts
// app/layout.tsx
import { Geist } from "next/font/google";
const geist = Geist({ subsets: ["latin", "greek"], variable: "--font-geist-sans" });
```

The Greek subset is **mandatory** (all user-facing strings are Greek per UI-06).

| Focus ring | `#0A0A0A` | `ring-2 ring-neutral-950 ring-offset-2` | Keyboard-visible focus on all interactive elements |

### Accent (Black) — Reserved-For List

The accent color `#000000` (black background / white text) is reserved EXCLUSIVELY for:

1. **Primary buttons** at rest — Add-to-cart "Προσθήκη", Copy-to-Messenger "📋 Αντιγραφή για Messenger", any future single primary CTA.
2. **Sticky cart button (FAB)** background at rest.
3. **Cart badge counter** (small black circle with white number on top of the sticky FAB).
4. **Active focus ring** outline color.

`text-black` (foreground) is permitted for headings and brand names — practically indistinguishable from `text-neutral-950` and used interchangeably; the contract treats them as the same accent token.

Accent is **NOT** used for: links (use `text-neutral-950 underline underline-offset-4` instead), dividers, card borders, badge text, hero text, hover states on cards, or filter chips (filter chips are Phase 2).

### 60 / 30 / 10 Audit

- **60% Dominant — white** — page background, card surfaces, drawer body, product page background.
- **30% Secondary — stone-50 + neutral grays** — hero band, footer subtle backgrounds, secondary text, soft borders/separators.
- **10% Accent — pure black** — primary CTAs + sticky FAB only.

Variant badge tints (emerald / amber / blue) are **semantic micro-accents** — they occupy <2% of pixels and do not violate the 60/30/10 budget because they map 1:1 to data (variant type), not chrome.

## Component Anatomy

Phase 1 ships exactly these UI surfaces. Each anatomy section lists structure, tokens used, and interaction states.

### 1. Hero Band (`<Hero />`)

**Where:** Top of `app/page.tsx` only (not on product pages).

**Structure:**
- Container: full-width band, `bg-stone-50`, vertical padding `py-10 md:py-12` (40px mobile, 48px tablet+), horizontal padding inherits page container.
- Inner content: centered, max-width prose, text-align center.
  - Wordmark: `"Marios Shop"` — `text-3xl md:text-4xl font-semibold text-neutral-950 leading-tight tracking-tight`.
  - Tagline: `"Επιλεγμένα αρώματα από τη συλλογή μου"` — `text-sm md:text-base text-neutral-600 mt-2`.
- Height: 140–180px on mobile, ~180–220px on desktop (governed by padding + content height, not fixed).

**States:** Static, no interaction.

### 2. Product Grid (`<ProductGrid />`)

**Where:** Below the hero on `app/page.tsx`.

**Structure:**
- Container: page-level `px-4 md:px-6 lg:px-8 max-w-7xl mx-auto py-6 md:py-8`.
- Grid: `grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4 md:gap-6`.
  - **Breakpoints (CAT-02):** mobile (<640px) 1 col, sm (≥640px) 2 col, md (≥768px) 3 col, lg (≥1024px) 4 col.

**Empty state (catalog never empty in Phase 1 since seed has 5–6 products):** Not required.

### 3. Product Card (`<ProductCard />`)

**Where:** Inside the product grid.

**Structure (CONTEXT.md D-16):**
- Wrapper: `<Link>` to `/product/[id]` wrapping the entire card. `display: block`.
- Card: `bg-white rounded-lg shadow-sm md:hover:shadow-md transition-shadow duration-150 overflow-hidden`.
- Image area: aspect ratio **1:1** (square — easier with mixed fragrance bottle proportions; final pick during impl per Claude's Discretion). `bg-stone-50` placeholder for image load. Uses `next/image` with `unoptimized` (D-23). `image_fallback_url` is the source for Phase 1 (D-03).
- Text block: `p-4 space-y-1`.
  - Brand: `text-base font-semibold text-neutral-950` (e.g., "Tom Ford").
  - Name: `text-sm text-neutral-600 leading-snug line-clamp-1` (e.g., "Tobacco Vanille").
  - Price row: `text-base font-semibold text-neutral-950 mt-2` formatted as `από {lowest_price}€` (Greek "from") OR just `{price}€` if a single variant. Use `lib/format.ts` `formatPrice(n)` → `${n}€` (D-26).
- Badges row: **below price**, `flex flex-wrap gap-1.5 mt-2`. One badge per distinct variant type present and in-stock. If all variants of one type are out of stock, do not render that type's badge.

**States:**
- Rest: `shadow-sm`.
- Hover (desktop only, `md:` and up): `shadow-md`.
- Focus (keyboard): `outline-none ring-2 ring-neutral-950 ring-offset-2 rounded-lg`.
- Active (tap): no separate style — Tailwind default browser tap highlight is acceptable (mobile-first; the link navigates immediately).

### 4. Variant Badge (`<VariantBadge type={...} />`)

**Where:** Product cards (badges row), product detail page (next to each variant row), and as type indicator in drawer line items.

**Structure (CONTEXT.md D-15):**

- Variants:

- Built on shadcn `<Badge variant="...">` with custom tone variants registered in `components/ui/badge.tsx`.

**States:** Static.

### 5. Product Detail Page (`app/product/[id]/page.tsx`)

**Structure:**
- Page container: `max-w-3xl mx-auto px-4 md:px-6 py-6 md:py-10`.
- Back link: top of page, `<Link>` with `text-sm text-neutral-600 hover:text-neutral-950 inline-flex items-center gap-1`, label `"← Πίσω στον κατάλογο"` (D-PROD-08).
- Hero image: full-width within container, `aspect-square md:aspect-[4/5] rounded-lg overflow-hidden bg-stone-50 mt-4 mb-6` with `next/image` and fallback handling.
- Title block:
  - Brand: `text-sm text-neutral-600` (e.g., "Tom Ford").
  - Product name: `text-xl font-semibold text-neutral-950 mt-1` (e.g., "Tobacco Vanille").
  - Line: `text-sm text-neutral-600 mt-1` (if `line` field present, e.g., "Private Blend").
- Notes: `text-sm text-neutral-600 mt-3` — comma-separated notes string OR an inline list. Single paragraph in Phase 1.
- Description: `text-sm text-neutral-950 leading-normal mt-4 whitespace-pre-line` — renders `description_gr`.
- **Variants list:** `<ul>` with `mt-8 space-y-3`. Each variant is a row (see anatomy below).

#### 5a. Variant Row (on Product Detail)

- Container: `flex items-center justify-between gap-3 py-3 border-t border-neutral-200`. First row keeps the border (uses `border-t`), giving a clean ruled list.
- Left cluster (vertical):
  - Top line: `flex items-center gap-2` — `<VariantBadge type={v.type} />` + size text `text-sm text-neutral-950` (e.g., "50ml"). For `opened` variants with `fill_pct`, append `· Γέμιση: {fill_pct}%` deferred to Phase 2 per PROD-07 → **not rendered in Phase 1**.
  - Bottom line: price `text-base font-semibold text-neutral-950` formatted `{price}€`.
- Right cluster: "Προσθήκη" button (see button anatomy below). When `stock === 0`, button is replaced by a disabled `<span>` showing "Εξαντλήθηκε" — see button anatomy state. (PROD-05 is mapped to Phase 2 in roadmap, but the seed data includes stock=0 per D-02; Phase 1 renders the disabled-stub affordance even if the polished message is Phase 2.)

### 6. Primary Button (`<Button variant="default">`)

shadcn `<Button>` with custom defaults. Two real button instances in Phase 1: **"Προσθήκη"** on product page, **"📋 Αντιγραφή για Messenger"** in cart drawer.

**Rest state:**
- `bg-black text-white text-sm font-semibold rounded-md px-4 h-11 inline-flex items-center justify-center gap-2`.
- Height 44px (touch target). Border-radius 6px (`rounded-md`).

**Hover (desktop):** `hover:bg-neutral-800` — slight lightening so the press is felt.

**Active (tap):** `active:bg-neutral-900 active:scale-[0.98] transition-transform duration-75`.

**Focus (keyboard):** `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-neutral-950 focus-visible:ring-offset-2`.

**Disabled (COPY-08, "Εξαντλήθηκε"):** `disabled:bg-neutral-200 disabled:text-neutral-400 disabled:cursor-not-allowed`. No hover/active styles. The "Εξαντλήθηκε" stub on a stock-0 variant uses the same disabled-button visual, label "Εξαντλήθηκε", `aria-disabled="true"`.

**Loading state:** Not required in Phase 1. Adds + copy are synchronous local operations. If clipboard write rejects, the error toast handles UX.

### 7. Sticky Cart Button (`<StickyCartButton />`)

**Where:** Fixed on every page (catalog + product detail). Always visible per D-12.

**Structure:**
- Container: `<button>` styled as `fixed bottom-4 right-4 z-40 h-14 w-14 rounded-full bg-black text-white shadow-lg shadow-black/15 inline-flex items-center justify-center`.
- Icon: lucide `<ShoppingBag size={22} />`.
- Badge counter: positioned `absolute -top-1 -right-1`, a `min-w-5 h-5 rounded-full bg-white text-black text-xs font-semibold inline-flex items-center justify-center ring-2 ring-black px-1`. Only renders when `itemCount > 0`. Shows the **total quantity** (sum of all `qty`), per CAT-09.
- Hidden until `isHydrated === true` to avoid SSR/client mismatch (D-07). Render a non-interactive placeholder (or `null`) until hydration to keep the visual position consistent.

**aria-label:** `"Άνοιγμα καλαθιού — {N} τεμάχια"` (dynamic). When empty: `"Άνοιγμα καλαθιού"`.

**States:**
- Rest: black, soft shadow.
- Hover (desktop): `hover:bg-neutral-800`.
- Active (tap): `active:scale-[0.95] transition-transform duration-100`.
- Focus: ring-2 ring-neutral-950 ring-offset-2 (white offset since FAB sits on white-ish page).

### 8. Cart Drawer (`<CartDrawer />`)

shadcn `<Sheet side="right">` configuration (D-09).

**Width:**
- Mobile: `w-[85vw]` (≈85% width).
- Tablet+ (`sm:`): `sm:w-[420px]` fixed.

**Layout — three vertical sections** using `flex flex-col h-full`:

#### 8a. Header
- `<SheetHeader>` with `px-5 py-4 border-b border-neutral-200`.
- Title: `"Καλάθι"` — `text-xl font-semibold text-neutral-950` via `<SheetTitle>`.
- Close button: shadcn default `<SheetClose>` (X icon, top-right). aria-label `"Κλείσιμο"`.

#### 8b. Item List (scrollable middle, `flex-1 overflow-y-auto px-5 py-4`)
- Empty state (if `items.length === 0`): see Empty state below.
- Otherwise: `<ul className="space-y-4">` with each item as `<DrawerItem />`.

##### 8b-i. Drawer Item (`<DrawerItem />`)

CART-06 fields: brand, name, variant (type + size), unit price, quantity, subtotal.

Structure: `flex gap-3 items-start`.

- Thumbnail (optional but recommended): 56×56px (`h-14 w-14 rounded-md bg-stone-50 overflow-hidden flex-shrink-0`) with the product image (fallback URL). Phase 1 includes the thumbnail.
- Right column (`flex-1 min-w-0 space-y-1`):
  - Brand: `text-sm font-semibold text-neutral-950 truncate` (e.g., "Tom Ford").
  - Name: `text-sm text-neutral-600 truncate` (e.g., "Tobacco Vanille").
  - Variant row: `flex items-center gap-2 text-xs text-neutral-600 mt-1` — `<VariantBadge />` + size text + `· {qty} × {unit_price}€`.
- Far-right column (`flex flex-col items-end gap-2`):
  - Subtotal: `text-sm font-semibold text-neutral-950` formatted `{subtotal}€`.
  - Remove button: small icon `<Trash2 size={16} />` inside a 44×44 tap area (`h-11 w-11 -m-3 inline-flex items-center justify-center text-neutral-400 hover:text-red-700 rounded-md focus-visible:ring-2 focus-visible:ring-neutral-950`), aria-label `"Αφαίρεση {brand} {name}"`.

**Phase 1 NO quantity stepper inside drawer** (D-11 + CART-07 is Phase 2). The quantity is read-only via "× N" in the variant row.

#### 8c. Footer (sticky bottom, `border-t border-neutral-200 px-5 py-4 space-y-3 bg-white`)

CART-09 fields: total €, total items.

- Summary row: `flex items-baseline justify-between`.
  - Left: `text-sm text-neutral-600` — `"Σύνολο"` + below it `text-xs text-neutral-600` — `"{N} τεμάχια"` (one item → "1 τεμάχιο", two+ → "N τεμάχια"; use `formatItemCount(n)`).
  - Right: `text-xl font-semibold text-neutral-950` — `{total}€`.
- Primary CTA: full-width Copy-to-Messenger button — `<Button className="w-full" />` (uses primary button anatomy from §6). Label `"📋 Αντιγραφή για Messenger"`. Disabled when `items.length === 0` (COPY-08).

#### 8d. Empty Cart State (inside Item List area)

- `flex flex-col items-center justify-center h-full text-center px-6 py-12`.
- Visual: lucide `<ShoppingBag size={32} className="text-neutral-400" />` (lone icon, no decoration).

- No CTA — the user is already in the drawer; closing it returns to the catalog.

**Drawer interaction states:**
- Open: backdrop `bg-black/40` (shadcn default).
- Close on backdrop click, Esc key, or X button.
- Focus trap inside the sheet (shadcn `Sheet` handles via Radix).
- Initial focus: the close button (Radix default for `Sheet`).
- Aria: `role="dialog"`, `aria-modal="true"`, `aria-labelledby="cart-title"` (`<SheetTitle id="cart-title">Καλάθι</SheetTitle>`).

## Accessibility Minimums (Phase 1)

Full a11y polish is Phase 3 (UI-09). Phase 1 floor:

- [ ] All interactive elements reachable with Tab and triggerable with Enter/Space.
- [ ] Visible focus ring on every interactive element (`focus-visible:ring-2 ring-neutral-950 ring-offset-2`).
- [ ] Semantic HTML: `<header>` for hero, `<main>` for page body, `<nav>` not needed (no nav in Phase 1), `<button>` for buttons (not `<div>`), `<ul>`/`<li>` for grid + drawer item list.
- [ ] Drawer is a Radix `Sheet` (built-in: `role="dialog"`, `aria-modal`, focus trap, Esc closes, restores focus to opener).

### Currency Format (D-26)

`lib/format.ts → formatPrice(n: number): string`

- Integer prices: `"180€"` (no decimals, no space, suffix €).
- Decimal prices: `"179.50€"` (dot decimal, two places only when needed). Phase 1 seed data uses integer prices only, but the formatter must handle decimals for future inventory edits.

### Pluralization

`lib/format.ts → formatItemCount(n: number): string`

- `n === 1` → `"1 τεμάχιο"`
- `n !== 1` → `"{n} τεμάχια"`

(Greek has only two grammatical numbers for this noun — singular and plural — so a binary check is correct.)

### Destructive Actions

**None in Phase 1.** "Καθαρισμός" (clear cart with confirm) is CART-10, mapped to Phase 2. Phase 1 only supports removing items one-by-one via the drawer remove icon — no confirmation step. (The remove icon turns red on hover as a soft visual warning but does not require confirmation; per spec a remove is single-item and immediately reversible by re-adding.)

### Copy-to-Messenger Format (COPY-02, COPY-03, COPY-05, D-25, D-26)

Generated by `lib/copy-format.ts → formatOrderText(cart): string`. Exact output for Phase 1:

```
Παραγγελία — Marios Shop

1. Tom Ford — Tobacco Vanille
   Σφραγισμένο 50ml — 180€ × 1 = 180€

2. Loewe — Bittersweet Oud
   Decant 5ml — 18€ × 2 = 36€

Σύνολο: 216€ — 3 τεμάχια
```

Rules:
- Header line: `"Παραγγελία — Marios Shop"` then blank line.
- Each item: index-dot-space + `"{brand} — {name}"` on first line, three-space indent + `"{TypeLabelGr} {size_ml}ml — {price}€ × {qty} = {subtotal}€"` on the indented line.
- Type labels are Greek (D-25): `Σφραγισμένο`, `Ανοιγμένο`, `Decant`. Phase 1 uses these (not the English "Sealed/Opened/Decant" shown in the user's earlier spec example).
- `fill_pct` is NOT included in copy text (D-25).
- Blank line between items.
- Footer: blank line then `"Σύνολο: {total}€ — {count} τεμάχια"` (use `formatItemCount` rule — singular `τεμάχιο` if 1).
